HTML

  • Hypertext Markup Language (HTML) is one of the three cornerstone languages used in web development.
  • HTML elements work together to build the structure of a webpage and provide a framework that makes it easy for a user to navigate the webpage.
  • The !DOCTYPE html declaration is letting the browser know to expect HTML on the page.
  • The html opening tag represents the root, or start, of our HTML document
  • The attribute of lang="en", tells the browser the language for the webpage.
  • The head element contains information, or metadata, about the webpage.
  • The title element defines the title for a webpage
  • The body element represents the visible content shown to the user.

CSS

  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • An external CSS style sheet is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • We can use the * selector when we want to apply a rule to all the elements visible on the page.
  • We can combine two selectors into a single CSS rule with a comma, allowing the declarations to affect both selectors.
  • Do Not Repeat Yourself (DRY) is a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained.
  • When we create an external CSS file, we need to link it to the HTML file in order for the styles to be dsiplayed on the application page.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds all of the changes in our working directory to the staging area of Git
  • git commit -m "*brief description of changes made*": commits the changes to our repository
  • git pull origin main: makes sure our current branch is up to date before opening a pull request
  • git push origin feature/*nameoffeaturebranch*: pushes up all of our changes to the remote github branch

JavaScript

  • JavaScript (JS) is a high-level dynamic programming language that allows users to interact with websites.
  • A variable is simply a named container that allows us to store data in our code.
  • To declare a variable, we use the 'var' keyword.
  • A string is contained within quotation marks.
  • We use an assignment operator = to assign the specific data that we want stored in our variable.
  • An array is a single variable that is used to hold a group of data.
  • Each line of JavaScript code ends with a semicolon.
